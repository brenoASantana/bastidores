# Contributing to BACKROOMS

Obrigado por considerar contribuir para o projeto BACKROOMS! Aqui estão as diretrizes para manter o projeto saudável e organizado.\n\n## Como Contribuir

### Reportar Bugs

Abra uma issue com:\n\n```\nTítulo: [Bug] Descrição curta\n\n## Comportamento Esperado\nO jogo deveria...\n\n## Comportamento Atual\nO jogo faz...\n\n## Para Reproduzir\n1. Faça X\n2. Faça Y\n3. Veja Z\n\n## Ambiente\n- Browser: Chrome 120\n- Sistema: Windows 11\n- Deploy: localhost / Vercel\n```\n\n### Sugerir Features\n\n```\nTítulo: [Feature] Título da sugestão\n\n## Descrição\nExplique a feature detalhadamente.\n\n## Caso de Uso\nPor que isso seria útil?\n\n## Implementação Sugerida (opcional)\nComo você poderia implementar?\n```\n\n### Pull Requests\n\n1. **Fork** the repository\n2. **Create branch**: `git checkout -b feat/feature-name`\n3. **Make changes** mantendo as conventions\n4. **Commit**: Use conventional commits\n5. **Push**: `git push origin feat/feature-name`\n6. **Open PR** com descrição clara\n\n#### Checklist para PR\n\n- [ ] Código segue as conventions do projeto\n- [ ] Testei a feature implementada\n- [ ] Não quebrei nenhuma feature existente\n- [ ] Atualizei a documentação relevante\n- [ ] Mensagem de commit é clara\n- [ ] Sem console.log() ou debug code\n- [ ] TypeScript está sem erros\n\n## Código\n\n### Style Guide\n\n**TypeScript/React:**\n```typescript\n// Use tipos explícitos\nconst calculateAnxiety = (delta: number, isSafe: boolean): number => {\n  return isSafe ? -0.15 * delta : 0.5 * delta\n}\n\n// Use nomes descritivos\nconst updatePlayerAnxietyLevel = () => {}  // ✅\nconst updateAnx = () => {}                // ❌\n\n// Use const por padrão\nconst MAX_ANXIETY = 100  // ✅\nlet anxiety = 0          // ✅ quando necessário\n\n// Use arrow functions\nconst handleClick = () => {}  // ✅\nfunction handleClick() {}     // ❌ em componentes\n```\n\n**React Components:**\n```typescript\n// Use 'use client' em client components\n'use client'\n\nimport { FC } from 'react'\n\ninterface Props {\n  anxiety: number\n  onZoneChange: (isOpen: boolean) => void\n}\n\nconst GameHUD: FC<Props> = ({ anxiety, onZoneChange }) => {\n  return (\n    <div>\n      {/* JSX */}\n    </div>\n  )\n}\n\nexport default GameHUD\n```\n\n**Tailwind:**\n```jsx\n// Agrupe classes\n<div className=\"absolute bottom-8 left-8 w-48 h-4 bg-gray-900 border border-gray-700\">\n\n// Evite inline styles\n<div style={{ color: 'red' }}>  // ❌\n<div className=\"text-red-500\">  // ✅\n```\n\n### Convenção de Nomes\n\n| Tipo | Padrão | Exemplo |\n|------|--------|----------|\n| Variáveis | camelCase | `let playerAnxiety` |\n| Constantes | UPPER_SNAKE_CASE | `const MAX_ANXIETY = 100` |\n| Funções | camelCase | `const calculateDelta = () => {}` |\n| Componentes | PascalCase | `const GameContainer = () => {}` |\n| Interfaces | PascalCase | `interface GameState {}` |\n| Métodos privados | _camelCase | `_initializeAudio()` |\n| Booleanos | isX, hasX, canX | `isInSafeZone`, `hasAudio` |\n\n### Commits\n\nUse Conventional Commits:\n\n```\nfeat(horror): add footstep event trigger\n\nAdiciona um novo evento assustador que dispara quando\nansiedade ultrapassa 60%. Sons de passos distantes\naumentam tension track.\n\nCloses #42\n```\n\n**Tipos:**\n- `feat:` - Nova feature\n- `fix:` - Bug fix\n- `refactor:` - Refatoração\n- `perf:` - Performance\n- `docs:` - Documentação\n- `style:` - Formatação\n- `test:` - Testes\n- `chore:` - Builds, deps\n\n## Documentação\n\nAtualize a documentação sempre que:\n- Adicionar nova feature\n- Mudar nome de variável/função pública\n- Alterar constants importantes\n- Adicionar novo sistema\n\n**Arquivos principais:**\n- `README.md` - Overview\n- `ARCHITECTURE.md` - Design técnico\n- `GAMEPLAY.md` - Mecânicas de jogo\n- `DEVELOPMENT.md` - Guia dev\n- Comentários `//` no código\n\n### Exemplo de Comentário\n\n```typescript\n// Sistema de Ansiedade\n// Calcula mudança de ansiedade baseado na zona\n// Zona segura: -0.15/s | Zona normal: +0.5/s | Zona aberta: +0.75/s\nconst calculateAnxietyDelta = (isInSafeZone: boolean): number => {\n  return isInSafeZone ? -ANXIETY_FALL_RATE : ANXIETY_RISE_RATE\n}\n```\n\n## Performance\n\n### Checklist de Performance\n\n- [ ] Sem console.log() em produção\n- [ ] Componentes memoizados quando necessário\n- [ ] useCallback para event handlers\n- [ ] Lazy loading de assets pesados\n- [ ] Sem re-renders desnecessários\n- [ ] Geometry e materiais otimizados\n\n### Medição\n\n```typescript\n// Use performance API\nconst start = performance.now()\n// ... operação ...\nconst duration = performance.now() - start\nconsole.log(`Duração: ${duration}ms`)\n```\n\n## Testing (Quando Implementado)\n\n```bash\n# Rodar testes\nnpm test\n\n# Com coverage\nnpm test -- --coverage\n```\n\n**Padrão AAA:**\n```typescript\ntest('calculates anxiety correctly', () => {\n  // Arrange\n  const delta = 0.016\n  const isSafe = false\n  \n  // Act\n  const result = calculateAnxietyDelta(delta, isSafe)\n  \n  // Assert\n  expect(result).toBe(0.5 * delta)\n})\n```\n\n## Code Review\n\n### Como Revisar\n\n1.
